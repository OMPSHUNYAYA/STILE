Truth vs transport

            
STILE does not deny transport. It separates delivery truth from delivery transport.

          

          

            
Layer 1
Structure Layer
Determines delivery truth from sender intent, receiver expectation, message identity, delivery context, and conflict state.

            
Layer 2
Representation Layer
Optional messages, APIs, receipts, logs, UI states, or certificates may represent the truth once it is structurally admitted.

            
Layer 3
Transport Layer
Optional networks, queues, gateways, retries, and acknowledgements may move data. They do not create correctness.

Break This STILE

            
The demo becomes meaningful when the invariant refuses to break.

          

          

            
1. Same structure, different result?
If identical structure produces a different visible delivery state or certificate, STILE fails. In this demo, identical structure always collapses to identical output.

            
2. Incomplete structure, forced delivery?
If missing identity, expectation, or context still produces delivery, safety fails. Here it resolves to ABSTAIN.

            
3. Conflict, arbitrary delivery?
If contradiction still admits delivery, correctness fails. Here it resolves to CONFLICT and no delivery appears.

          

        


        

          

            
Reference kernel logic

            
The HTML demo mirrors the minimal Python idea: derive flags, resolve structure, then reveal delivery only after alignment.

          

          
resolve(structure) ->

RESOLVED   if structure_aligned
ABSTAIN    if structure is incomplete
CONFLICT   if structure is inconsistent

where:

structure_aligned = complete AND consistent
message_delivered iff structure_aligned
same structure -> same delivery -> same certificate
