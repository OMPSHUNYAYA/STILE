Note:
For aligned structure, reason_codes is an empty array [].
It is shown as NONE in this plain-text verification file for readability.

4. Structural alignment rule

structure_aligned = complete AND consistent
message_delivered iff structure_aligned

5. Expected behavior

complete AND consistent -> RESOLVED -> delivery visible
incomplete structure -> ABSTAIN -> no delivery
inconsistent structure -> CONFLICT -> no delivery
identical structure -> identical delivery state and certificate

7A. Structural Truth vs Network Illusion

This test simulates a traditional delivery model where:
delivery is assumed based on send/ack logic instead of structure.

In browser console:

function fakeNetworkDelivery(s) {
  return {
    message_id: s.message_id,
    status: "DELIVERED",
    via: "NETWORK_SIMULATION"
  };
}

Test mismatch case:

let structural = await resolveStructure(scenarios[1].s)
let network = fakeNetworkDelivery(scenarios[1].s)

console.log("STRUCTURAL:", structural.delivery)
console.log("NETWORK:", network)

Expected:

STRUCTURAL: null
NETWORK: { message_id: "MSG-001", status: "DELIVERED", via: "NETWORK_SIMULATION" }

Test conflict case:

structural = await resolveStructure(scenarios[2].s)
network = fakeNetworkDelivery(scenarios[2].s)

console.log("STRUCTURAL:", structural.delivery)
console.log("NETWORK:", network)

Expected:

STRUCTURAL: null
NETWORK: { message_id: "MSG-001", status: "DELIVERED", via: "NETWORK_SIMULATION" }

Observation:

network-based model reports delivery even when structure is invalid

structural model refuses delivery when structure is not aligned

Conclusion:

delivery != transmission
delivery != acknowledgement
delivery = resolve(structure)
